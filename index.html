Environment Setup and Logging Workflow

  1. System Environment
  • The analysis was conducted on a local machine configured with the necessary resources for running Elasticsearch and Kibana.
  • All services were hosted locally, and the soclog index was created to store collected security logs.
  1. Data Sources and Log Collection
  • Security-relevant logs were ingested into Elasticsearch, including authentication attempts, malware detections, file access, and connection attempts.
  • Logs were normalized to include key fields: @timestamp, action, ip, threat, and user.
  1. Tools Used
  • Elasticsearch: Core data storage and indexing engine for log analysis.
  • Kibana: Visualization and querying interface for exploring the soclog index.
  • Data Visualizer: Used to analyze field distributions, top values, and generate insights on user behavior, threat types, and source IPs.
  1. Querying and Filtering
  • Specific ES queries were written to detect security events:
  • Failed login attempts
  • Malware detections, including Trojans, Rootkits, and Ransomware
  • Connection attempts from known IP addresses
  • Combined queries for detecting suspicious activity across actions and threats
  1. Analysis Workflow
  • Data exploration using the Discover panel in Kibana.
  • Identification of suspicious actors, IP addresses, and event types using top-value statistics for each field.
  • Sorting and filtering logs by @timestamp to prioritize recent events.
  • Screenshots were captured for reporting purposes, highlighting key events and trends.
  1. Monitoring and Threat Detection
  • Failed logins were monitored as potential indicators of brute-force or credential-stuffing attacks.
  • Malware events were analyzed by type (Trojan, Rootkit, Ransomware) and correlated with user and source IP.
  • Connection attempts and file access events were investigated for unauthorized or unusual activity.

SECTION 1: Failed Login Attempt Detection

Query Used: action: "login failed"

Description: This query identifies failed authentication attempts recorded in the system logs. These events may indicate unauthorized access attempts or early stages of account compromise.

Findings: Multiple failed login attempts were observed from different IP addresses targeting several user accounts within a short time window.

Severity Rating: Medium

SOC Assessment
Failed login attempts are considered early indicators of compromise and are monitored to detect brute-force or credential misuse activities.

SECTION 2: Malware Detection Events
